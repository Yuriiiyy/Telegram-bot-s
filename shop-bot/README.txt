====================================================================
GODLY's Shop — Telegram-магазин одягу з мікросервісною архітектурою
====================================================================

[ ПРО ПРОЄКТ ]
Цей проєкт є готовим рішенням для автоматизації продажів одягу через Telegram.
Він побудований на базі мови Python та бібліотеки pyTelegramBotAPI (telebot).


[ ОСОБЛИВОСТІ ТА АРХІТЕКТУРА ]
Головна перевага проєкту — розділення логіки на два незалежні модулі (боти),
які працюють паралельно в одному середовищі:

1. Клієнтський бот (main.py):
   Повноцінний каталог товарів, інлайн-меню для вибору розмірів, адмін-панель
   для швидкого додавання позицій та покроковий скрипт збору даних покупця.

2. Адмін-сервіс сповіщень (order_bot.py):
   Окремий мікробот, який приймає замовлення від клієнтського бота та миттєво
   пересилає їх менеджеру. Це повністю розвантажує основний інтерфейс клієнта 
   і запобігає збоям чи крашам через флуд або перевищення лімітів Telegram API.


[ БЕЗПЕКА ТА ВАЛІДАЦІЯ ДАНИХ (ANTI-CRASH) ]
- Регулярні вирази (модуль 're'): 
  Реалізовано жорстку перевірку введених користувачем даних. Телефон перевіряється 
  строго за маскою (10 цифр), а ПІБ — на наявність лише літер. Бот автоматично 
  відсікає некоректні введення.
  
- HTML-екранування (html.escape): 
  Усі текстові введення покупців проходять фільтрацію. Навіть якщо користувач 
  введе спецсимволи (*, _, `, "), які зазвичай ламають синтаксис Telegram API, 
  бот обробить їх абсолютно безпечно і не вилетить із помилкою "Bad Request".
  
- Прямий зв'язок (Deep Linking): 
  В адмін-панель замовлення приходять із клікабельним посиланням на профіль клієнта 
  через його Telegram ID (tg://user?id=). Це дозволяє менеджеру написати покупцю 
  в один клік, навіть якщо у того прихований @username.


[ СТЕК ТЕХНОЛОГІЙ ]
- Мова розробки: Python 3.10+
- Ключова бібліотека: pyTelegramBotAPI (v4.26.0)
- Додаткові модулі: re (RegEx), html (безпека розмітки)


[ ЯК ЗАПУСТИТИ ПРОЄКТ ]
1. Склонуйте або скопіюйте всі файли проєкту в одну папку.
2. Створіть двох ботів у @BotFather та отримайте їхні токени.
3. Дізнайтеся свій Telegram ID (наприклад, через @userinfobot).
4. Відкрийте файли main.py та order_bot.py й вставте ваші токени та ID у 
   відповідні константи на початку файлів.
5. Встановіть необхідну бібліотеку через термінал:
   pip install -r requirements.txt
6. Запустіть головний файл програми:
   python main.py

Після цього обидва боти почнуть працювати паралельно.
